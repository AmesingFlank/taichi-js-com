Textures

Textures are an important concept in computer graphics. They are images that come with hardware-accelerated sampling and filtering, and can be used to add details to 3D geometry. This page documents texture support in taichi.js.

For examples of complete taichi.js programs that make use of textures, see

  • Fractal Cloth
  • GLTF Physically-Based Rendering

Creating Textures​

A standard 2D texture with RGBA components can be created using

let texture = ti.texture(4, [width, height]);

The argument 4 specifies the amount of components in each texel. You can also pass 1 here for red-only, or 2 for red and green only.

Loading Texture from An Image​

You can create an image from either an ImageBitmap, an HTMLImageElement, or simply from an url of the image. You can use these APIs:

let texture = await ti.Texture.createFromBitmap(bitmap);
let texture = await ti.Texture.createFromHtmlImage(htmlImageElement);
let texture = await ti.Texture.createFromURL(url);

Sampling from a Texture​

Given a texture object, you can sample from it in a kernel:

let texel = ti.textureSample(tex, texCoords);

Here, the first argument is the texture object itself (either exposed to kernel scope via ti.addToKernelScope(..) or passed as ti.template() argument). The 2nd argument is the texture coordinates, which should be a 2D vector for 2D textures, or a 3D vector for 3D textures and cubemaps.

3D Textures​

3D textures can be created by providing an extra depth value when calling ti.texture:

let texture = ti.texture(4, [width, height, depth]);

Cubemaps​

A cubemap is a collection of six 2D textures, each representing one side of a cube. In computer graphics, cubemaps are often used to represent an environment map. Read more about cubemaps here. In taichi.js, you can create a cubemap using an array of six ImageBitmaps, HTMLImageElements, or urls:

let cubemap = await ti.CubeTexture.createFromURL([
    urlRight,
    urlLeft,
    urlTop,
    urlBottom,
    urlFront,
    urlBack,
]);

Depth Textures​

As previously mentioned, a depth buffer can be created by

let depthBuffer = ti.depthTexture(renderTarget.dimensions);

In render pipelines, you may use depth buffers for depth testing by specifying

ti.useDepth(depthBuffer);

But for algorithms such as Shadow Maps, you can explicitly carry out "compare-and-sample" operations on depth buffers:

let visibility = ti.textureSampleCompare(shadowMap, coords, depth);

Render Targets​

Previously we showed how you can create a render target texture which corresponds to an HTML canvas:

let htmlCanvas = document.getElementById('result_canvas');
let renderTarget = ti.canvasTexture(htmlCanvas);

But you can also create render targets that are not tied to any HTML canvases. This is helpful if your renderer contains multiple passes, and you need renderTargets to store intermediate results (e.g. GBuffers). You can do this directly by declaring a texture and using it as the render output in your pipeline:

let gBuffer = ti.texture(4, [width, height]);
...
let render = ti.kernel((..) => {
    ti.clearColor(gBuffer, ...);
    for (let v of ti.inputVertices(...)) {
        ...
    }
    for (let f of ti.inputFragments()) {
        ti.outputColor(gBuffer, ...);
    }
})

MSAA Textures​

MSAA, or Multi-Sampled Anti Aliasing, helps reduce jagged edges at the boundaries of rendered objects. In taichi.js, you can enable MSAA by providing an optional sample count at the end of your render targets and depth buffers. In the following example, the renderTarget, gBuffer, and depth texture, all have 4xMSAA:

let htmlCanvas = document.getElementById('result_canvas');
let renderTarget = ti.canvasTexture(htmlCanvas, 4);
let depthTexture = ti.depthTexture( [htmlCanvas.width, htmlCanvas.height], 4);
let gBuffer = ti.texture(4, [htmlCanvas.width, htmlCanvas.height], 4);

Reading and Writing Texels​

taichi.js allows you to read and write the values of individual texels, given the integer coordinates of the texel. These can be achieved using the following functions:

let texelValue = ti.textureLoad(tex, texelCoords);
ti.textureStore(tex, texelCoords, texelValue)

As an example, the Fractal Cloth program uses a parallel compute pipeline to populate a texture using ti.textureStore (..), and samples the texture in a fragment shader. Similarly, in a more advanced scenario, the GLTF Physically-Based Rendering program uses a compute pipeline to pre-filter environment maps represented as equi-rectangular textures, and sample them during rendering.
